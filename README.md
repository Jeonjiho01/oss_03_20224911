#  리눅스 프로세스 관리 명령어: top, ps, jobs, kill 분석

본 문서는 리눅스/유닉스 시스템에서 프로세스 상태를 모니터링하고 제어하는 데 사용되는 핵심 명령어인 `top`, `ps`, `jobs`, `kill`에 대해 정리합니다. (참고 자료: 유닉스 명령어, 에디터, make)

---

## 1. top (프로세스 모니터링)

`top` 명령어는 시스템의 **실시간** 상태를 표시하며, 현재 실행 중인 프로세스 목록을 CPU 사용률, 메모리 사용량 등의 기준으로 정렬하여 보여줍니다.

> **ℹ️ 용도:** 시스템 성능 모니터링 및 자원 사용률이 높은 프로세스 식별에 유용합니다.

| 주요 표시 정보 | 설명 |
| :--- | :--- |
| **PID** | 프로세스 ID (고유 번호) |
| **USER** | 프로세스를 실행한 사용자 |
| **%CPU** | CPU 사용률 (백분율) |
| **%MEM** | 물리 메모리 사용률 (백분율) |
| **COMMAND** | 실행 중인 명령어 (프로그램 이름) |

## 2. ps (프로세스 상태 확인)

`ps` 명령어는 특정 시점의 **프로세스 스냅샷**을 보여줍니다. [cite_start]`top`이 실시간이라면, `ps`는 현재의 정지된 목록을 보는 것과 같습니다[cite: 466].

> **💡 중요 옵션:** 시스템 전체의 상세한 정보를 확인하기 위해 `-ef` 또는 `-aux` 옵션을 주로 사용합니다.

| 옵션 | 설명 |
| :--- | :--- |
| **-a** | [cite_start]**숨겨진 파일**(.bashrc 등)을 포함한 **모든 파일**을 리스트 [cite: 195] |
| **-l** | [cite_start]파일에 관한 **더 상세한 정보**를 출력 [cite: 178, 481] |
| **-e** | [cite_start]**시스템 전체** 프로세스를 보여줌 [cite: 482] |
| **-u user_name** | [cite_start]특정 사용자(`user_name` 또는 **UID**)의 **모든 프로세스**를 보여줌 [cite: 483, 485]

### ps 실행 결과의 주요 필드

| 필드 | 설명 |
| :--- | :--- |
| **PID** | [cite_start]**프로세스의 고유 ID 번호** [cite: 476] |
| **TTY** | [cite_start]프로세스가 수행 중인 **터미널 번호** [cite: 475] |
| **S (STATE)** | [cite_start]프로세스의 **현 상태** (R: 현재 실행 중, S: 입력 대기 중 등) [cite: 474] |
| **TIME** | [cite_start]사용한 **총 CPU 시간** [cite: 473] |
| **COMMAND** | [cite_start]실제 그 **프로세스의 내용** [cite: 472] |

## 3. jobs (백그라운드 작업 관리)

`jobs` 명령어는 **현재 쉘(Shell)** 에서 실행 중이거나 중지된 **백그라운드 작업**의 목록을 보여줍니다.

> [cite_start]**💡 백그라운드 실행:** 명령어 끝에 `&`를 붙이면 프로그램을 **백그라운드**로 실행할 수 있습니다. [cite: 503]
> ```bash
> $ sleep 60 & 
> ```
> `jobs` 목록에는 백그라운드로 실행된 작업과 `Ctrl+Z`로 중지된 작업이 표시됩니다.

## 4. kill (프로세스 강제 종료)

`kill` 명령어는 지정된 **PID**를 가진 프로세스에게 **시그널**을 보내 프로세스를 제어합니다. [cite_start]주로 프로세스를 **강제 종료**할 때 사용됩니다[cite: 492].

* **용법:** `kill [-시그널번호] PID`
* **PID**는 프로세스 ID 번호로, `ps` 명령어로 확인할 수 있습니다.

### ⚠️ 강제 종료 시그널

[cite_start]대부분의 경우 프로세스를 **강제 종료**할 때 **`-9`** 옵션 (SIGKILL)을 사용합니다[cite: 493].


## 한줄 정리

**`top`, `ps`, `jobs`, `kill`** 명령어는 리눅스에서 **프로세스 상태를 확인(`top`, `ps`), 백그라운드 작업을 관리(`jobs`), 불필요하거나 문제 있는 프로세스를 종료(`kill`)** 하는 데 사용되는 핵심 유틸리티입니다.



```bash
$ kill -9 42823 
# [cite_start]PID 42823 프로세스를 강제로 죽임 [cite: 494, 515, 516]
